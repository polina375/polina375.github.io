## Цель работы
Освоение основ работы с платформой SourceCraft: создание организации и репозитория, настройка CI/CD для автоматической сборки статического сайта на базе MkDocs, публикация сайта и обеспечение его обновления при каждом изменении в репозитории.

## Задание
Создать публичную организацию и репозиторий на SourceCraft.

Сгенерировать персональный токен доступа (PAT) с правами Maintainer.

Добавить SourceCraft как второй удалённый репозиторий в локальный проект.

Настроить CI/CD-воркфлоу для автоматической сборки MkDocs-сайта.

Создать конфигурацию публикации (.sourcecraft/sites.yaml), указав ветку и папку с готовым сайтом.

Разрешить возможные конфликты при синхронизации веток.

Проверить, что при пуше в main запускается сборка и сайт обновляется.

## Нюансы
Токен доступа – создаётся один раз и отображается только при создании; необходимо сохранить его в надёжном месте.

Файл .sourcecraft/sites.yaml должен быть в корне главной ветки (main) и содержать корректные путь к собранному сайту и имя ветки для публикации.

Триггер воркфлоу – в файле воркфлоу обязательно указывать on: push: branches: main, иначе автоматический запуск не произойдёт.

Конфликт веток – при рассинхронизации локальной и удалённой веток необходимо выполнить git pull --rebase или git push --force (только если вы уверены в своих изменениях).

Сборка в отдельную папку – рекомендуется собирать сайт в папку site (или другую, не занятую исходниками), чтобы не затирать исходные файлы.
